# FixForesight Backend API Contract

This document serves as the official API contract and source of truth for the FixForesight Predictive + Prescriptive Maintenance Platform.

---

## Endpoints

### 1. GET /machines
Retrieves the list of all monitored machines with their latest live vitals.
* **Response Status**: `200 OK`
* **Response Body**:
```json
[
  {
    "machine_id": "M101",
    "machine_name": "CNC Spindle Unit",
    "status": "Warning",
    "temperature": 75.0,
    "pressure": 30.0,
    "vibration": 4.8,
    "rpm": 2500
  },
  {
    "machine_id": "M102",
    "machine_name": "Hydraulic Press",
    "status": "Healthy",
    "temperature": 48.5,
    "pressure": 120.0,
    "vibration": 0.8,
    "rpm": 1200
  }
]
```

---

### 2. GET /predictions
Retrieves the failure probability predictions generated by the predictive maintenance models (e.g. TensorFlow).
* **Response Status**: `200 OK`
* **Response Body**:
```json
[
  {
    "machine_id": "M101",
    "failure_probability": 82,
    "predicted_failure": "Bearing Failure",
    "time_to_failure": "5 Days"
  },
  {
    "machine_id": "M102",
    "failure_probability": 5,
    "predicted_failure": "Normal Operation",
    "time_to_failure": "N/A"
  }
]
```

---

### 3. GET /recommendations
Retrieves the prescriptive maintenance recommendations matching predicted failures.
* **Response Status**: `200 OK`
* **Response Body**:
```json
[
  {
    "machine_id": "M101",
    "recommendation": "Schedule bearing replacement immediately. Flush grease system and inspect rotor alignment.",
    "priority": "High",
    "confidence": 91
  }
]
```

---

### 4. GET /alerts
Retrieves the live events/alerts feed received via SNS topics.
* **Response Status**: `200 OK`
* **Response Body**:
```json
[
  {
    "alert_id": 1,
    "machine_id": "M101",
    "severity": "Critical",
    "message": "Bearing failure risk exceeds 80%",
    "created_at": "2026-06-22T15:00:00Z"
  }
]
```

---

### 5. GET /analytics
Retrieves global machine health counts for dashboard indicators (e.g. pie/gauge charts).
* **Response Status**: `200 OK`
* **Response Body**:
```json
{
  "healthy": 85,
  "warning": 10,
  "critical": 5
}
```

---

### 6. GET /search
Queries historical incident logs indexed in Apache Solr.
* **Request Query Parameters**:
  * `q` (string, search query. Default: `*:*`)
* **Response Status**: `200 OK`
* **Response Body**:
```json
{
  "numFound": 1,
  "docs": [
    {
      "id": "inc-001",
      "machine_id": "M101",
      "failure_signature": "Vibration levels spike (bearing wear), high temperature near main drive spindle",
      "action_taken": "Replaced rotary bearing B-10 and adjusted rotor alignment",
      "outcome": "Resolved",
      "date": "2026-03-15T08:00:00Z"
    }
  ]
}
```

---

# Database Schema Reference

### machines
* `machine_id` (VARCHAR PRIMARY KEY)
* `machine_name` (VARCHAR)
* `status` (VARCHAR)
* `temperature` (DOUBLE PRECISION)
* `pressure` (DOUBLE PRECISION)
* `vibration` (DOUBLE PRECISION)
* `rpm` (INT)
* `created_at` (TIMESTAMP)

### predictions
* `prediction_id` (SERIAL PRIMARY KEY)
* `machine_id` (VARCHAR REFERENCES machines(machine_id))
* `failure_probability` (DOUBLE PRECISION)
* `predicted_failure` (VARCHAR)
* `time_to_failure` (VARCHAR)
* `created_at` (TIMESTAMP)

### recommendations
* `recommendation_id` (SERIAL PRIMARY KEY)
* `machine_id` (VARCHAR REFERENCES machines(machine_id))
* `recommendation` (TEXT)
* `priority` (VARCHAR)
* `confidence` (DOUBLE PRECISION)
* `created_at` (TIMESTAMP)

### alerts
* `alert_id` (SERIAL PRIMARY KEY)
* `machine_id` (VARCHAR REFERENCES machines(machine_id))
* `severity` (VARCHAR)
* `message` (TEXT)
* `created_at` (TIMESTAMP)
